# NeuroStack -- Generative AI & Backend Developer Internship Hackathon Guide

## Overview

This project is part of the **NeuroStack GenAI Build Sprint**, a 7‑day
challenge to build a **production‑ready mini Generative AI application**
using only **free tools**.

The objective is to demonstrate:

-   AI engineering capability
-   Backend development
-   Problem‑solving skills
-   Product thinking
-   Deployment and documentation

Participants must build a complete working system with frontend,
backend, GenAI workflow, and deployment.

------------------------------------------------------------------------

# Project Goal

Build a **GenAI application** consisting of:

-   **React.js frontend**
-   **FastAPI or Django backend**
-   **Signup / Login authentication**
-   **LangChain-based GenAI pipeline**
-   **RAG (Retrieval Augmented Generation) workflow**
-   **Deployment on Hugging Face Spaces**
-   **Complete README documentation**

------------------------------------------------------------------------

# Mandatory Tech Stack

Frontend: - React.js

Backend: - FastAPI or Django

AI Framework: - LangChain

Vector Database: - FAISS or Chroma

Deployment: - Hugging Face Spaces

Models: - Open‑source models from Hugging Face - Llama - Mistral - Gemma

------------------------------------------------------------------------

# Free‑Tools Only Rule

Allowed:

-   Hugging Face open‑source models
-   LangChain
-   FAISS / Chroma
-   React.js
-   FastAPI / Django
-   Hugging Face Spaces deployment

Not Allowed (Disqualification):

-   OpenAI API
-   Any paid API or model
-   Paid cloud services
-   Paid vector databases
-   Any service requiring a credit card

------------------------------------------------------------------------

# Selected Problem Statement

## SaaS Support Copilot (RAG System)

Users often struggle to search through long documentation.

This project builds an **AI support assistant** that retrieves relevant
information from documentation and generates grounded answers.

------------------------------------------------------------------------

# Dataset Requirement

Create a dataset containing:

-   20--50 FAQ or documentation items

Stored in:

-   JSON file

------------------------------------------------------------------------

# RAG Pipeline Architecture

The system must include:

1.  Document ingestion
2.  Text chunking
3.  Embedding generation
4.  Vector database storage
5.  Hybrid retrieval
6.  Context validation
7.  Response generation

Components:

-   Embeddings
-   FAISS / Chroma vector database
-   Hybrid retrieval (BM25 + embeddings)
-   Answer validation layer

------------------------------------------------------------------------

# Hallucination Prevention

Implement safeguards such as:

-   Minimum similarity score threshold
-   Reject low‑context queries
-   Rephrase prompts
-   Return fallback response when context is insufficient

Example fallback message:

"Sorry, I couldn't find a relevant answer in the documentation."

------------------------------------------------------------------------

# Required UI Features

The UI must contain **three main pages**.

------------------------------------------------------------------------

# 1. Login / Signup Page

Initial page of the application.

### Desktop Layout

Left side: - Welcome message

Right side: - Login card

Example text:

Welcome Back

Fields: - Email - Password

Button: - Login

Link: - Signup

### Signup Animation

When the user clicks **Signup**:

-   Card slides **from right to left**
-   Signup form appears on the **left**
-   Message appears on the **right**

Signup fields:

-   Name
-   Email
-   Password

------------------------------------------------------------------------

# Mobile Layout

Top: - Welcome message

Below: - Login / Signup form

The layout should be fully **responsive**.

------------------------------------------------------------------------

# 2. Home Page

After login, the user first sees a **Hero Section**.

### Hero Section

Features:

-   Large headline
-   Short description
-   Stylish UI

Bottom left corner:

Scroll indicator:

"Scroll Down"

### Special Behavior

When scrolling:

-   Hero section disappears
-   Main page loads **in the same screen space**
-   User cannot scroll back to hero

Hero appears again only after:

-   Page refresh
-   New login
